## Часть 1 — Simplify

Прочитай каждый файл. Упрости:

- **Over-engineering:** лишние абстракции, преждевременные обобщения, классы там где хватит функции, фабрики для одного варианта
- **Дублирование:** повторяющийся код → вынести в функцию (только если 3+ повторений)
- **Сложные условия:** вложенные if/else → early return, guard clauses
- **Лишние обёртки:** промежуточные переменные используемые один раз, тривиальные функции-прокси
- **Избыточные типы:** TypeScript типы которые дублируют очевидное из контекста

## Часть 2 — Cleanup

После simplify — почисти:

- `console.log` / `console.debug` для отладки (НЕ трогать production logging)
- Закомментированный код (`// old implementation`, `/* disabled */`)
- `TODO` / `FIXME` / `HACK` без номера тикета
- Избыточные комментарии которые описывают очевидное
- Неиспользуемые импорты
- Пустые строки в избытке (3+ подряд → 1)

## НЕ трогать

- Экспортируемые интерфейсы — сигнатуры остаются
- Поведение кода — результат работы не меняется
- Файлы за пределами списка
- JSDoc/TSDoc на экспортируемых функциях
- Комментарии с бизнес-логикой или edge-case пояснениями
- `TODO`/`FIXME` с номером тикета
- Existing code не изменённый в рамках задачи

## Формат ответа

```
SIMPLIFIED:
- <файл>: <что упростил и почему>

CLEANED:
- <файл>: <что удалил> (N строк)

NO_CHANGES:
- <файл>: код чистый
```

Если всё чисто — `NO_CHANGES_NEEDED`.
