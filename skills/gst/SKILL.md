# Статус разработки

Показать текущее состояние git-репозитория: что изменилось, что закоммичено, где сконцентрирована работа.

---

## Алгоритм

### Шаг 1 — Собрать данные

Запусти `git-data-collector` через Agent tool.

Агент определён в `${CLAUDE_PLUGIN_ROOT}/skills/gst/agents/git-data-collector.md`.

Задача агенту:

```
Собери данные о текущем состоянии git-репозитория.
```

Дождись результата. Агент вернёт структурированный блок с секциями:
`BRANCH`, `WORKING_TREE`, `DIFF_VS_DEFAULT`, `COMMITS`, `HOT_FILES`, `STASH`.

### Шаг 2 — Форматировать вывод

На основе данных от агента сформируй отчёт по шаблону ниже.

```
Ветка: <branch> [ahead N, behind M origin/<upstream>]

Изменения: +<added> -<removed> строк | <N> файлов
   staged: <N>  unstaged: <N>  untracked: <N>

Изменённые файлы (vs main):
   <директория>/
     <статус> <файл> (+N -M)
     <статус> <файл> (+N)

Горячие файлы:
   1. <path> — +N -M
   2. <path> — +N -M
   3. <path> — +N -M

Коммиты (от main): +N коммитов
   <hash> <message> — <время>
   <hash> <message> — <время>
   <hash> <message> — <время>

Stash: N записей

Сводка: <2-3 предложения — что сделано, на основе коммитов и диффов>
```

### Шаг 3 — Правила форматирования

- Относительные таймстемпы: "2ч назад", "вчера", "3 дня назад". Без абсолютных дат.
- Группировать файлы по директориям, не алфавитный список.
- Статус файла: `M` изменён, `A` добавлен, `D` удалён, `R` переименован.
- В сводке описывать что сделано ("добавлена авторизация через JWT"), не перечислять файлы.
- Пропускать пустые секции. Если нет stash — не показывать секцию. Если нет горячих файлов — не показывать.
- Без эмодзи. Только текстовые заголовки.

### Шаг 4 — Граничные случаи

**Detached HEAD:**
Вместо имени ветки показать хэш и ближайший тег или ветку:

```
Ветка: detached at abc1234 (рядом с v1.2.0)
```

**Нет upstream:**
Показать ветку без ahead/behind:

```
Ветка: feature/auth (нет upstream)
```

**На основной ветке (main/master):**
Diff vs main пуст, коммитов от merge-base 0. Показать последние 5 коммитов с пометкой:

```
Последние коммиты:
   <hash> <message> — <время>
```

Пропустить секции "Изменённые файлы" и "Горячие файлы".

**Merge conflicts:**
Выделить в отдельную секцию перед остальным выводом:

```
КОНФЛИКТЫ СЛИЯНИЯ:
   <файл1>
   <файл2>
```

**Пустой репозиторий:**
Показать только untracked файлы:

```
Новый репозиторий. Untracked файлов: <N>
```

**Нет изменений:**
Показать ветку, последние коммиты, сводку. Пропустить секции "Изменения", "Изменённые файлы", "Горячие файлы".
